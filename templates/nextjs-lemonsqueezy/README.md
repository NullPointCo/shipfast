A production-ready **Next.js 14 (App Router) + Lemon Squeezy** SaaS starter.

## Features

- Next.js 14 App Router + TypeScript
- Lemon Squeezy hosted checkout via `/api/checkout` (no card UI to build)
- Lemon Squeezy webhook handler at `/api/webhooks` (HMAC-signed, verified)
- Pricing page with client-side checkout redirect
- Dockerfile for containerized deploy (Render / Fly / any OCI host)
- `.env.example` documenting every required variable

## Quick start

```bash
npm install
cp .env.example .env.local   # fill in your Lemon Squeezy keys + variant IDs
npm run dev                  # http://localhost:3000
```

## Lemon Squeezy setup

1. Create a store at <https://app.lemonsqueezy.com> and add two subscription
   products (e.g. Starter, Pro). Copy each **Variant ID**.
2. Create an API key (Settings > API) and a webhook (Settings > Webhooks)
   pointing at `https://<your-domain>/api/webhooks`. Copy the secret.
3. Fill `.env.local`:
   - `LEMONSQUEEZY_API_KEY`
   - `LEMONSQUEEZY_STORE_ID`
   - `LEMONSQUEEZY_WEBHOOK_SECRET`
   - `NEXT_PUBLIC_LS_STARTER_VARIANT_ID`
   - `NEXT_PUBLIC_LS_PRO_VARIANT_ID`
4. For local webhooks, use the official CLI:
   ```bash
   npx lemon-squeezy-webhooks listen --forward-to localhost:3000/api/webhooks
   ```

## Deploy

```bash
docker build -t {{PROJECT_NAME}} .
docker run -p 3000:3000 --env-file .env.local {{PROJECT_NAME}}
```

Set the same env vars on your host (Render/Fly). Point `NEXT_PUBLIC_BASE_URL`
at your live domain.
