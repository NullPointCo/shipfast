## Features

- Next.js 14 App Router + TypeScript
- Supabase Auth (email/password) via `@supabase/ssr`
- Server-side session refresh in `middleware.ts` (no flicker, secure cookies)
- SSR auth in Server Components — read the logged-in user on the server
- Protected `/dashboard` route (Server Component redirects unauthenticated users)
- Sign up / log in / sign out flows
- Dockerfile for containerized deploy (Render / Fly / any OCI host)
- `.env.example` documenting every required variable

## Quick start

```bash
npm install
cp .env.example .env.local   # fill in your Supabase project URL + anon key
npm run dev                  # http://localhost:3000
```

## Supabase setup

1. Create a project at [supabase.com](https://supabase.com).
2. In **Project Settings → API**, copy:
   - `Project URL` → `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` key → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. (Optional) Under **Authentication → Providers**, enable email confirmations.
   With confirmations on, new users receive a magic link before they can sign in.

## How auth works here

- `lib/supabase/client.ts` — browser client for client components (forms).
- `lib/supabase/server.ts` — server client for Server Components / Actions.
- `lib/supabase/middleware.ts` — refreshes the session cookie on every request.

This is the official Supabase-recommended `@supabase/ssr` pattern.
