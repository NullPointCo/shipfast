# {{PROJECT_NAME}} — Astro + Stripe SaaS Starter

A production-ready Astro 4 starter with Stripe **subscription** checkout and
signed webhooks. Server-rendered via the Node adapter so the API routes work
in production.

## Features
- ⚡ Astro 4 (App-like routing, fast static shell)
- 💳 Stripe Checkout (subscription mode) with promotion codes
- 🔐 Signed webhook verification (`STRIPE_WEBHOOK_SECRET`)
- 🐳 Dockerfile for one-command deploys
- 🧩 Pricing page wired to `import.meta.env.PUBLIC_STRIPE_*_PRICE_ID`

## Quick start
```bash
cp .env.example .env        # fill in your Stripe test keys + price IDs
npm install
npm run dev                 # http://localhost:3000
```

## Configure Stripe
1. Create two **recurring** Prices in the Stripe Dashboard (Products → Prices).
2. Paste their IDs into `.env` as `PUBLIC_STRIPE_STARTER_PRICE_ID` and
   `PUBLIC_STRIPE_PRO_PRICE_ID`.
3. For webhooks locally: `stripe listen --forward-to localhost:3000/api/webhooks`
   and copy the `whsec_...` value into `STRIPE_WEBHOOK_SECRET`.

## API routes
| Route | Method | Purpose |
|-------|--------|---------|
| `/api/checkout` | POST | Create a Checkout session, returns `{ url }` to redirect to. Body: `{ priceId }`. |
| `/api/webhooks`  | POST | Verify + handle Stripe events (`checkout.session.completed`, `invoice.payment_failed`). |
